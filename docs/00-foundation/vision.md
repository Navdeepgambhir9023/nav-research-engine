## Purpose

Defines what the Research Operating System (ROS) *is* and *is not*. Serves as the primary anchor for all architectural decisions and scope discussions.

## Audience

- All contributors
- Stakeholders
- Reviewers evaluating project scope

## Content

### What the ROS Is

The **nav-research-engine** is a Research Operating System designed to autonomously discover, organize, validate, and prioritize business opportunities within the Web3 ecosystem.

The ROS is:
- **A living knowledge platform** — The repository itself becomes an evolving intelligence asset, not just a container for code
- **A modular research engine** — Composable components that work together to process, analyze, and generate research
- **An evidence-driven system** — Every conclusion is traceable to verifiable sources and passes quality gates
- **A human-in-the-loop architecture** — Human oversight is a core design requirement, not an afterthought
- **A self-improving system** — Each execution cycle produces outputs that improve future cycles

### What the ROS Is NOT

The ROS explicitly excludes:

- **An AI chatbot** — No conversational interface; all interactions are structured workflows
- **An AI wrapper** — Not a simple wrapper around LLM APIs; the intelligence is in the orchestration and knowledge structures
- **A prompt collection** — Not a library of prompts; prompts are generated dynamically based on system state
- **An automation script** — Not a linear sequence of steps; operates as a stateful system with autonomous decision-making
- **A data aggregation tool** — Not merely collecting data; transforming raw information into actionable intelligence

### Core Value Proposition

> The repository is the product. The research engine is the operating system that maintains it.

This inverted relationship means that:
1. The quality of outputs improves over time as the knowledge base grows
2. The system becomes more valuable as it operates, not just as code is written
3. Contributors can be researchers, not just engineers
